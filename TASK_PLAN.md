# Travel Expense Tracker - 開発タスクプラン

このドキュメントは、Travel Expense Trackerアプリの開発タスクを管理するためのものです。
各タスクは細かく分割されており、チェックボックスで進捗を管理できます。

**凡例**:
- [ ] 未着手
- [x] 完了
- [~] 進行中（オプション：手動でマーク）

## Phase 1: バックエンド（Supabase）🗄️

### 1.1 データベーススキーマの作成

#### Users テーブル
- [ ] `users` テーブルの作成
  ```sql
  - id: UUID (Primary Key)
  - email: String
  - display_name: String
  - profile_image_url: String
  - onboarding_completed: Boolean (Default: false)
  - created_at: Timestamp
  - updated_at: Timestamp
  ```
- [ ] Supabase Auth と連携するトリガーの作成

#### Trips テーブル
- [ ] `trips` テーブルの作成
  ```sql
  - id: UUID (Primary Key)
  - user_id: UUID (Foreign Key → users)
  - name: String
  - destination: String
  - start_date: Date
  - end_date: Date
  - currency_code: String
  - currency_symbol: String
  - total_budget: Decimal (nullable)
  - cover_image_url: String (nullable)
  - notes: Text (nullable)
  - created_at: Timestamp
  - updated_at: Timestamp
  ```
- [ ] インデックスの作成（user_id, start_date）

#### Categories テーブル
- [ ] `categories` テーブルの作成
- [ ] デフォルトカテゴリーのシードデータ作成

#### Tags テーブル

#### Category_Budgets テーブル

### 1.2 Row Level Security (RLS) の設定
- [ ] `users` テーブルのRLSポリシー
  - [ ] 自分のデータのみ読み取り可能
  - [ ] 自分のデータのみ更新可能
- [ ] `trips` テーブルのRLSポリシー
  - [ ] 自分の旅行のみCRUD可能
- [ ] `expenses` テーブルのRLSポリシー
  - [ ] 自分の旅行に紐づく支出のみCRUD可能
- [ ] `categories` テーブルのRLSポリシー
  - [ ] デフォルトカテゴリーは全員読み取り可能
  - [ ] 自分のカスタムカテゴリーのみCRUD可能
- [ ] `tags` テーブルのRLSポリシー
  - [ ] 自分のタグのみCRUD可能
- [ ] その他テーブルのRLSポリシー

### 1.3 ストレージの設定

- [ ] ストレージのRLSポリシー設定
